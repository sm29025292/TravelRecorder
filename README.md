# TravelRecorder 旅遊記錄

把原本用 Google Sheets / Excel 記錄的「旅遊行程 + 花費」搬成一個**本機優先（local-first）、可離線的 PWA 網頁 App**。
所有金額計算（換匯、小計、總計、平均、分帳）都是程式邏輯的**純函式**，**不會再因為新增 / 移動格子而公式壞掉**。

- 🧮 **計算自動且穩定**：改匯率，所有花費與行程的台幣金額即時重算。
- 🗂️ **景點庫**：景點集中管理、依**國家／都市／區域**三層樹狀分組，可標類型（景點／美食／住宿／交通）並級聯篩選；已排入任何旅程的景點會標 ✓（已去過）。
- 🧭 **行程**：依旅程日期自動列出**每一天**，同日依開始時間排序；每列可填**開始／結束時間**，時數自動計算；小計以外幣為主、下方灰字顯示台幣；一鍵**複製逐日行程**成純文字貼到通訊軟體。
- 🧾 **花費 + 分帳**：登記同行者，花費每列可選付錢者與分攤對象（預設均分），自動算每人結餘與「誰付給誰」，**各幣別獨立結算、不換匯、不跨幣別對沖**；標「已結清」的列不列入結算。
- 📱 **可離線、可裝到手機**：響應式 (RWD) + PWA，手機加到主畫面後離線也能用。
- 💾 **資料在本機**：存在瀏覽器 (IndexedDB)，用 JSON 匯出／匯入做備份與換機，**可選「取代」或「合併」**。
- 🔒 **預設無後端、無帳號**：資料不會上傳到任何伺服器；如需跨裝置，可 **opt-in** 手動上傳**加密**備份到自己的私人 GitHub Gist。

## 功能

- **旅程清單**：建立多個旅程，各自設定日期、國家／都市、外幣代碼／名稱、匯率。國家從景點庫連動下拉，選好國家可自動帶入幣別與匯率（優先取其他旅程最近用過的匯率）。
- **總覽**：旅程基本資料與匯率設定。
- **花費**：機票 / 飯店 / 保險 / 購物等大項，每列可選幣別（外幣或台幣）+ 手續費，自動算小計、總計、平均（平均分母＝分帳成員數）。原本的「購物」分頁已併入花費。
- **行程（逐日）**：
  - 自動依旅程起訖日期列出每一天（含週幾與當日小計）。
  - 景點下拉來自景點庫，依旅程國家自動鎖定，可再依類型／都市篩選。
  - 開始／結束時間 → 自動算時數；同日依開始時間升冪排序。
  - 每列小計以外幣為主、下方灰字顯示台幣；頁尾整趟總計顯示「小時 · 外幣 · 台幣」。
  - 一鍵**複製逐日行程文字**（不含金額）貼到通訊軟體。
- **分帳**：登記同行者；花費每列可選「付錢」與「分攤對象」（預設全體均分，可改成個人或子集），自動算出每人「已付／應分攤／結餘」與**成對淨額**的「誰付給誰」（兩兩互欠對沖、可追溯）；**各幣別獨立結算**（TWD 桶最前、外幣列的台幣手續費入 TWD 桶）；未指定付錢者或標「已結清」的列不列入結算。
- **行李**：可勾選的打包清單（項目 / 份量 / 備註）。**新建旅程時會自動從最近更新的旅程複製一份行李清單**（勾選狀態歸零，可自行增刪）。
- **景點庫**：
  - 依**國家／都市／區域**三層樹狀分組（未分類節點置頂），可標類型（景點／美食／住宿／交通），支援類型／國家／都市／區域級聯篩選與名稱搜尋。
  - **匯入 CSV**（合併不覆蓋，支援「國家／地點／區域」欄自動辨識與向下填滿）。
  - **整理重複**：名稱正規化分桶、可選保留者、一鍵合併並自動重指行程參照。
  - **健檢**：偵測孤兒行程列（景點已刪除仍被行程參照），可整批清除參照。
  - **搬移／編輯**：節點層可批次改子樹地點；單列可搬移到別的位置。
  - 網址欄旁 ↗ 一鍵開新分頁；priority 以三顆星呈現、可點擊調整。
- **備份**：右上角「匯出備份／匯入」；匯入時可選「合併（保留本機並用備份補上）」或「取代（清空後以備份為準）」。
- **同步（opt-in）**：右上角「同步」，把加密備份手動上傳／下載到自己的私人 GitHub Gist。前端 AES-GCM 256bit + PBKDF2-SHA256（310k iterations）加密，PAT 只存本機、無第三方伺服器。

## 開發與建置

```bash
npm install        # 安裝相依套件
npm run dev        # 本機開發（預設 http://localhost:5173）
npm run test       # 執行單元測試（金額 / 匯入 / 分組 / 分帳 等純函式）
npm run build      # 型別檢查 + 產生靜態網站到 dist/
npm run preview    # 預覽 build 後的網站
```

> PWA 圖示由 `node scripts/gen-icons.mjs` 產生（已內附，無需重跑）。

## 部署到 GitHub Pages（自動、免費）

已內附 GitHub Actions 工作流程 `.github/workflows/deploy.yml`：**每次推送到 `main` 分支**就會自動
`npm ci` → `npm run build` → 發佈到 GitHub Pages。

一次性設定：

1. 把開發分支**合併到 `main`**（工作流程設定在 `main` 觸發）。
2. **Settings → Pages → Build and deployment → Source** 選「**GitHub Actions**」。
3. **Settings → Environments → github-pages → Deployment branches and tags** 把 `main` 加進允許清單（或設「No restriction」），否則 deploy job 會被環境保護擋下。
4. 完成後網址約為 `https://<你的帳號>.github.io/<repo 名稱>/`
   （例：`https://sm29025292.github.io/TravelRecorder/`）。

專案已設定 `base: './'` 與 `HashRouter`，放在子路徑或重新整理都不會 404。
也可改丟到 Vercel / Netlify / 自己的空間，或本機直接 `npm run preview`。

## 存取控制 / 隱私

- 發佈到 GitHub Pages 的網站**是公開的**（個人帳號無法做檢視權限控制，那需要 GitHub Enterprise Cloud）。
- **但你的資料不會外洩**：本 App 預設沒有後端，資料只存在「每個人自己瀏覽器」的 IndexedDB，
  陌生人打開公開網址只會看到一個空白 App，看不到任何人的旅程或花費。
- **同步（opt-in）** 也不經第三方伺服器：資料在瀏覽器端以 AES-GCM 加密後直接寫入使用者**自己**的私人 Gist；GitHub PAT 只存在本機 localStorage，密語可選擇不記住。
- 若未來真的想「只有特定人能打開 App」，最佳免費解法是改用
  **Cloudflare Pages + Cloudflare Access**（Zero Trust 免費版可達 50 人，用 Google / Email 驗證碼登入、白名單指定 email）。

## 資料與備份

- 資料只存在「目前這個瀏覽器」的 IndexedDB；**換瀏覽器 / 換手機 / 清除瀏覽資料前，請先「匯出備份」或用「同步」上傳到 Gist**。
- 備份檔是一個 JSON，含所有旅程、景點、花費、行程、成員、行李；用「匯入」可在新裝置還原，並自動相容舊版景點欄位（`region` → `country`/`city`/`district`、舊 `shopping` 表自動搬進 `expenses`）。
- 匯入時可選：
  - **合併**（預設建議）：以備份為準覆蓋同 id 資料，本機獨有的資料保留。
  - **取代**：清空本機所有資料、完全以備份內容取代。
